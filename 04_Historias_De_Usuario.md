# **Historias de Usuario**

## Definición
Las historias de usuario describen funcionalidades desde la perspectiva del usuario.

Formato:
Como [rol], quiero [acción], para [beneficio].

---

## 1. Autenticación y Seguridad

- HU-01: Como usuario, quiero autenticarme de forma segura, para proteger mi información.
- HU-02: Como administrador, quiero gestionar usuarios, para controlar el acceso.
- HU-03: Como administrador, quiero definir roles, para restringir permisos.
- HU-04: Como administrador, quiero bloquear accesos tras intentos fallidos, para evitar ataques.

---

## 2. Acceso Ciudadano

- HU-05: Como ciudadano, quiero acceder a datos sin iniciar sesión, para informarme fácilmente.

---

## 3. Gestión Ambiental

- HU-06: Como técnico, quiero validar datos automáticamente, para evitar errores.
- HU-07: Como encargado del aire, quiero calcular promedios de contaminantes, para cumplir normativa.
- HU-08: Como encargado de agua, quiero clasificar su calidad, para evaluar su uso.
- HU-09: Como encargado de residuos, quiero validar datos según normativa, para asegurar cumplimiento.
- HU-10: Como encargado vehicular, quiero registrar emisiones, para controlar contaminación.

---

## 4. Monitoreo y Visualización (GIS)

- HU-14: Como usuario, quiero activar capas del mapa, para analizar datos específicos.
- HU-15: Como ciudadano, quiero ver calidad del aire en el mapa, para conocer el estado ambiental.
- HU-16: Como planificador, quiero ver rutas de basura, para mejorar la gestión.
- HU-17: Como técnico, quiero ver mapas de ruido, para identificar zonas críticas.

---

## 5. Reportes

- HU-18: Como directivo, quiero generar reportes automáticos, para tomar decisiones.
- HU-19: Como usuario externo, quiero ver indicadores actualizados, para transparencia.

---

## 6. Integración de Sistemas

- HU-20: Como administrador, quiero sincronizar sensores, para validar datos en tiempo real.
- HU-21: Como analista, quiero extraer datos automáticamente, para analizarlos.
- HU-22: Como técnico, quiero recibir alertas de errores, para corregir fallos.

---

## 7. Interoperabilidad

- HU-23: Como encargado, quiero integrar datos con otros sistemas, para mantener consistencia.
- HU-24: Como supervisor, quiero importar rutas GPS, para visualizarlas en tiempo real.
- HU-25: Como inspector, quiero reportar incidentes automáticamente, para agilizar procesos.

---

## 8. Denuncias

- HU-30: Como técnico, quiero recibir denuncias automáticamente, para atenderlas rápido.
- HU-31: Como ciudadano, quiero ver el estado de mi denuncia, para hacer seguimiento.

---

## 9. Normativa y Control

- HU-36: Como operador, quiero validar datos según normativa, para cumplir la ley.
- HU-37: Como directivo, quiero ver indicadores globales, para decisiones estratégicas.
- HU-38: Como inspector, quiero usar firma digital, para validez legal.
- HU-40: Como técnico, quiero acceso restringido, para evitar errores.
- HU-41: Como auditor, quiero ver registros de cambios, para control total.

---

## Observación
Las historias de usuario deben estar vinculadas a requerimientos funcionales (RF) y servir como base para pruebas y desarrollo ágil.
